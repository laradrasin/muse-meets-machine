# Project Index

This repository is a public-facing index of selected builds from Muse Meets Machine and Message & Muse.

Some working systems are private because they contain personal workflow architecture, client-adjacent methodology, unpublished writing systems, or connected account details. This index documents the public version: what was built, why it matters, and what each build is teaching me about AI-assisted communication work.

## AI Operating System / Claude OS

**Status:** Private workflow architecture; public summary only.

A private AI operating system connecting Claude projects, Notion, Google Drive, Google Calendar, Gmail, Canva, and structured project contexts into one adaptive workflow system.

The core idea is simple: instead of using AI as a one-off answer machine, I use it as a connected working layer that can orient itself across projects, files, workflows, and recurring tasks.

Public notes from this build focus on the architecture and communication design decisions, not the private implementation details.

### What it explores

- Context as working infrastructure
- AI as a coordination layer across tools
- Project instructions as operating design
- Documentation as memory
- Human judgment as the routing layer

## Voice Calibration Methodology

**Status:** Public methodology note available here: [Voice Calibration Methodology](voice-calibration-methodology.md).

A repeatable method for preserving authentic human voice in AI-assisted writing workflows.

The process starts with a deliberately simple exercise: generate AI drafts, have the human writer revise them, then analyze the difference between the draft and the revision. The “delta” becomes the voice reference.

The point is not to make AI sound more human in the abstract. The point is to make the human writer’s actual judgment, rhythm, texture, and preferences more explicit.

### What it explores

- Voice as a system, not just a style
- AI-assisted writing without voice collapse
- Agent-ready editorial infrastructure
- How to keep the human in the loop in a meaningful way

## Product Evaluation Rubric

**Status:** Internal methodology/tooling; public summary only.

A 40-item product assessment framework built as both an interactive tool and a Claude skill.

The rubric turns product and messaging judgment into a structured evaluation process, covering positioning, trust, product clarity, competitive viability, messaging quality, red flags, and overall gut check.

### What it explores

- Turning tacit judgment into structured methodology
- Product evaluation as a communication problem
- Messaging quality as part of product trust
- Frameworks as both diagnostic tools and teaching tools

## Content Engine / Morning Digest

**Status:** Internal workflow; public summary only.

An AI-assisted content research and strategy workflow designed around a clear division of labor: AI surfaces relevant inputs and asks better questions; the human owns the actual point of view.

The system is not designed to outsource voice. It is designed to reduce the blank-page problem while keeping authorship and judgment with the person writing.

### What it explores

- AI as research assistant, not ghostwriter
- Minimum-effective-dose content systems
- Thought leadership without content-farm behavior
- Protecting human reaction, taste, and timing
